# LABELS.md

Labels recomendados para el flujo GitHub Issues + Codex.

## Flujo Codex

- `codex-ready`: issue listo para que Codex lo trabaje.
- `codex-working`: Codex esta trabajando este issue.
- `codex-review`: cambio listo para revision o prueba manual.
- `codex-blocked`: issue bloqueado por informacion faltante, dependencia o prueba externa.

## Seguridad y validacion

- `protected-change`: autoriza tocar un sistema protegido si el issue incluye `[Protected Change]`.
- `roblox-tested`: cambio probado manualmente en Roblox Studio.
- `do-not-touch-pasture`: recordatorio explicito de no tocar Pasture.

## Areas del juego

- `combat`: combate, disparo, dano, proyectiles o BattleMode.
- `homestead`: terreno, casa, granja o sistemas de homestead.
- `storage`: cofre, inventario, EggStorage o carga de municion.
- `ui`: interfaces, HUD, botones o pantallas.

## Tipo de trabajo

- `bug`: correccion de comportamiento incorrecto.
- `feature`: funcionalidad nueva o mejora de gameplay.
